Why VoiceIME

VoiceIME 不是单点 AI 小工具，而是一层可持续使用的输入工作流。

真正影响日常效率的，不是某个模型多强，而是你能否在不离开当前上下文的前提下，快速完成采集、理解、生成和输出。 VoiceIME 选择从菜单栏、快捷键和统一能力层入手，把这些动作收敛成一套每天都愿意用的桌面体验。

少切换

不再在网页、录音、截图、朗读工具之间来回跳

同一个产品里完成语音、文本、回复、OCR 和朗读动作，减少窗口切换和心智打断。

更高频

以快捷键和菜单栏为主入口

高频动作不需要先打开厚重界面，适合聊天沟通、邮件处理、文档整理等持续发生的任务。

更灵活

既支持在线服务，也支持本地路线

HTTP provider 与本地 runtime 并存，让你可以按连接条件、隐私要求和体验偏好自由组合。

Capabilities

核心能力

VoiceIME 把高频 AI 输入能力整合成一套统一产品面，而不是把每一项能力做成彼此割裂的孤岛。

Voice Input

语音输入

按住快捷键即可录音，松开后继续完成转录、润色与文本注入。支持 HTTP STT，也支持本地 Local / SenseVoice 路线。

Smart Reply

智能回复

多角色并发生成回复，可快速对比不同语气、身份与语言风格，适合聊天、邮件、工单与跨语种沟通。

Smart Speech

智能发声

支持 Edge TTS 与 Local / VoxCPM，既能朗读当前文本，也能结合参考音频完成更接近真实使用的发声链路。

Smart OCR

智能OCR

截图取字、二维码识别、图片/PDF 内容提取之后，可以直接进入后续编辑、回复与处理链路。

Quick Launcher

快速入口

统一搜索面板直达历史记录、工具魔盒、设置与核心动作，不需要反复记忆深层入口。

History

历史追溯

语音历史、回复历史、发声历史与 OCR 历史分组清晰，方便复盘输入来源、生成结果与输出轨迹。

Scenarios

适合哪些真实场景？

它不是只适合 demo 演示，而是适合每天反复发生、又最容易被工具切换打断的工作。

聊天与即时沟通

语音转文字、快速生成回复、切换语气与语言风格，适合微信、Slack、Telegram、飞书等高频沟通场景。

邮件与工单处理

把草稿整理、翻译、回复建议和正式文本输出串起来，减少在多个工具间粘贴与反复润色。

文档与知识整理

把截图 OCR、语音记录、文本编辑和历史追溯放进一个统一工作台，适合写作、归档与知识沉淀。

朗读与语音输出

支持直接朗读，也支持更复杂的发声与虚拟麦克风路线，适合演示、内容输出与辅助表达场景。

Advantages

为什么它比碎片化工具组合更顺手？

因为 VoiceIME 关注的不是“功能数量”，而是功能之间如何自然衔接，最终能否形成真正可日用的桌面体验。

统一入口

菜单栏 + 快捷键作为统一入口，降低功能发现和反复进入配置页的成本。

统一心智

输入、理解、生成、输出和历史回看都在同一产品语义下发生，减少“每个功能一个世界”的割裂感。

统一路由

同一套产品既可连接在线服务，也可走本地 runtime，适合不同环境下的可用性与隐私需求。

统一追溯

历史记录与下载分发保持清晰，让长期使用而不是一次性体验成为可能。

FAQ

常见问题

如果你考虑把 VoiceIME 当作日常输入层，通常会先关心这些问题。

VoiceIME 和普通语音输入工具有什么不同？

它不是只做语音转文字，而是把语音输入、智能回复、智能发声、智能OCR 与历史追溯放在同一个菜单栏工作流中。

它适合哪些人使用？

适合高频聊天、邮件回复、文档整理、截图取字、内容朗读和需要快速切换 AI 输入能力的 macOS 用户。

是否支持本地模型路线？

支持。VoiceIME 同时兼容 HTTP 路线与本地 runtime，例如本地 STT 与 Local / VoxCPM，让你可以按环境和偏好自由选择。

首次安装后通常要做什么？

完成安装后，按系统提示授予录音、辅助功能、屏幕录制等必要权限；这样语音输入、截图 OCR 与文本注入才能稳定工作。

如果我只是想先体验一下，入口会不会很重？

不会。VoiceIME 以菜单栏与快捷键为主入口，适合从单个能力开始试用，再逐步把更多工作流收进日常使用。
